### Data acquisition — completed

The merged CasJobs query returned 507,249 rows. CasJobs caps exports at ~500 MB so the dataset came back in three chunks, concatenated with a bash header/tail trick before loading into pandas.

Bulk downloads completed for the full dataset:
- **Images**: 478,388 Legacy Survey JPEGs (94% of 507k rows — ~29k 404s)
- **Spectra**: 466,798 SDSS FITS files (92% of 507k rows — ~40k 404s)

After applying Galaxy Zoo vote filters (nvote_tot≥20, p_dk<0.30) and the redshift cut (spec_z≤0.10, recommended by Galaxy Zoo 1 for reliable morphological resolution) and label threshold (vote fraction >0.60), the usable *labeled* subset shrinks to **214,383 rows**. Of those labeled rows, ~66% have a matching spectrum on disk — this intersection (labeled + image + spectrum) forms the **working set of 142,383 rows** used for spectral and fusion model training. The other 34% of labeled rows can still be used for tabular and image training, which is planned for the next phase.

### Label audit and preprocessing decisions

Ran a full column audit on the 723-column wildcard export. Found four profile log-likelihood columns (`lnLDeV_r`, `lnLExp_r`, `lnLDeV_g`, `lnLExp_g`) with 67–84% sentinel values (-9999, meaning the profile fit failed). Dropped these — a feature that is 75%+ imputed-to-median carries almost no signal. This revised the feature set from the Week 1–2 plan.

Label distribution at threshold=0.60, z≤0.10, nvote_tot≥20, p_dk<0.30:

|---|---|---|
| Elliptical | 108,296 | 50.5% |
| Spiral | 104,613 | 48.8% |

Merger count (1,474) exceeded the 500-example minimum so the threshold stayed at 0.60. Working set (has image AND spectrum): 142,383 rows, split 70/15/15 to give a fixed test set of 32,202 galaxies used across all four models.

### Tabular baseline — XGBoost

Trained XGBoost on tabular photometric and spectroscopic features (fracDeV_r, colour indices, concentration, velocity dispersion, eigenspectrum PCA coefficients, etc.). Class-weighted loss to handle merger imbalance.

| Metric | Score |
|---|---|
| Macro F1 | 0.759 |
| Accuracy | 93.4% |
| Merger F1 | 0.396 |

Runs in under a minute on CPU. The speed is because XGBoost uses gradient boosting on the raw feature table — no image loading, no neural network. The merger F1 of 0.396 reflects class imbalance: XGBoost can separate ellipticals and spirals cleanly using fracDeV_r and colour g-r, but mergers are morphologically diverse and overlap with both classes in feature space.

---

### Image baseline — ResNet-18

ResNet-18 pretrained on ImageNet, final linear layer replaced with a 3-class head. Trained with AdamW, CosineAnnealingLR, class-weighted cross-entropy, early stopping on val macro F1.

GPU utilisation was initially very low (~20–25%) due to JPEG loading bottlenecking the data pipeline. Fixed by increasing batch size to 256, using 6 DataLoader workers with persistent workers and prefetch_factor=4.

| Metric | Score |
|---|---|
| Macro F1 | 0.806 |
| Accuracy | 95.7% |
| Merger F1 | 0.492 |

Best single-modality model. Visual spatial structure in the JPEGs carries information beyond what the photometric summary statistics capture.

---

### Spectral baseline — 1D-CNN

Three-layer 1D-CNN (32→64→128 filters, global average pooling, linear head). Spectra are resampled to a uniform 2700-bin grid (3800–9200 Å, 2 Å/px), clipped at ±5σ, median-normalised. Zero-filled for the ~34% of galaxies with no downloaded FITS file.

The main engineering challenge: loading 142k FITS files via astropy at startup was taking ~3 MB/s. Fixed by preloading all spectra into a shared-memory numpy array using 32 parallel threads (ThreadPoolExecutor), reducing startup from ~15 minutes to ~2 minutes.

| Metric | Score |
|---|---|
| Macro F1 | 0.563 |
| Accuracy | 77.7% |
| Merger F1 | 0.118 |

Lowest macro F1. This is expected: the 1D-CNN sees the integrated spectrum, which correlates with star-formation rate (a physical property) rather than spatial morphology directly. Spiral galaxies with suppressed star formation can look spectroscopically elliptical, and vice versa. The spectral model is best understood as a *physical type* classifier, not a morphological one — which sets up an interesting stretch experiment (see next steps).

---

### Late fusion — multi-modal model

Concatenated ResNet-18 (512d) + CNN1D (128d) + tabular MLP (64d) + has_spectrum flag (1d) = 705d vector → Linear(256) → Linear(3). Two-phase training:
- Phase 1: freeze image and spectral encoders, train fusion head + tabular MLP only (10 epochs). Pre-computed encoder embeddings once into RAM to avoid re-running frozen encoders every batch.
- Phase 2: unfreeze all parameters, joint fine-tuning at LR=1e-5, early stopping patience=5.

Had a Windows-specific issue: Phase 2 DataLoader workers (spawned via `multiprocessing.spawn`) were failing to load `cublas64_12.dll` due to pagefile exhaustion. Resolved by sharing the pre-loaded spectra array as a torch shared-memory tensor so workers only need to load JPEG images from disk.

Best checkpoint at Phase 2 epoch 15 (val macro F1 = 0.866).

| Metric | Score |
|---|---|
| Macro F1 | **0.865** |
| Accuracy | **96.5%** |
| Merger F1 | **0.657** |

## Challenges encountered

**CUDA zombie process (Windows):** After one training run completed, the process held 9.6 GB of VRAM without releasing it. Subsequent training attempts hung at CUDA initialisation. Diagnosed via `nvidia-smi`, resolved by force-killing the stale process via PowerShell. Now verifying `nvidia-smi` is clean before starting each GPU run.

**Windows pagefile exhaustion:** `multiprocessing.spawn` fails to load torch DLLs when the pagefile is too small. Affects any code that spawns workers after VRAM is partially allocated. Workaround: `num_workers=0` for test evaluation; shared-memory tensors for Phase 2 training.

## What's next (Weeks 5–6)

The pipeline works, but the current results use a restricted working set (142k rows — only galaxies with both image and spectrum). This leaves ~72k labeled galaxies unused for tabular and image training. The next phase addresses this and strengthens the justification for all design choices.

**Full data training.** Tabular and image models will be retrained on the full 214k labeled set (test set unchanged for fair comparison), showing how much the spectral coverage constraint costs.

**Hyperparameter tuning per model.** Rather than just reporting results from the default settings, each model will be evaluated across a grid of key hyperparameters:
- XGBoost: grid search over n_estimators × max_depth × learning_rate (~54 combinations, 5-fold CV, ~15 min)
- ResNet-18: 4 configurations varying LR and dropout (~6 hr GPU)
- 1D-CNN: 4 configurations varying LR and filter sizes (~2 hr GPU)

This turns the architecture choices from assertions into evidence.

**Fusion ablation study.** Three additional fusion runs: image+tabular only, spectral+tabular only, image+spectral only. Quantifies the contribution of each individual modality to the fusion model and directly justifies the three-modality design.

**Interpretability.** XGBoost feature importance plot (validates that fracDeV_r and colour g-r rank highest) and GradCAM activation maps on the image model (shows what spatial features ResNet-18 actually uses).

**Physical classification stretch task.** Re-label galaxies as star-forming vs passive using g-r colour as a proxy, then compare tabular and spectral baselines on this task. The hypothesis is that spectra should significantly outperform tabular here — the spectral model is naturally suited to physical type classification, even if it underperforms on morphological classification.
