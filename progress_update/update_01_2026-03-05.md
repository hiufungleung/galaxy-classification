## What I've done (Weeks 1–2)

The proposal scheduled Weeks 1–2 as *Project Planning & Literature Review* and the start of *Data Acquisition & Label Generation*. I've finished the planning work and made solid progress on data acquisition infrastructure.

**Data sources confirmed.** The three modalities are locked in: tabular photometry from SDSS DR19 (CasJobs), galaxy images from the DESI Legacy Survey, and spectra from the SDSS DR19 archive. The plan is exactly as proposed — XGBoost/MLP for tabular, ResNet-18 for images, 1D-CNN for spectra, combined via late fusion.

**SQL pipeline sorted out.** Hit an unexpected snag with CasJobs: running a wildcard query (`p.*, s.*, v.*`) on 10,000 rows returned 723 columns where 15 were silently renamed `Column1`–`Column14`. The two critical ones were `SpecObj.z` (spectroscopic redshift, renamed to `Column8`) clashing with `PhotoObj.z` (z-band magnitude), and `SpecObj.mjd` clashing with `PhotoObj.mjd`. Resolved by auditing all 723 columns, dropping ~650 that are instrument artefacts, database admin, or redundant, and rewriting the query with explicit column names and SQL aliases (`spec_z`, `spec_mjd`). The final query selects ~75 columns and should come in well under the 500 MB quota.

**Feature selection done.** Identified the key photometric features: `fracDeV_r` (strongest single predictor — 1.0 = elliptical, 0.0 = spiral), colour indices from extinction-corrected magnitudes (`dered_g - dered_r`), concentration (`petroR90_r / petroR50_r`), axis ratios, and profile fit likelihoods. Spectroscopic features include redshift, velocity dispersion, and eigenspectrum PCA coefficients (`theta_0–4`).

**Labelling strategy decided.** Using threshold-based labels (vote fraction > 0.60) rather than argmax, which discards ambiguous cases. Also filtering out rows with < 20 votes or > 30% "don't know" votes. Three classes: elliptical, spiral, merger. Expecting ~55–60% spiral, ~35–40% elliptical, ~1–3% merger — will use class-weighted loss for the imbalance.
